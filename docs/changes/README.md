# Change Log

Record of changes made to the RunRoute codebase.

## Format

Each change note should be a separate file named: `YYYY-MM-DD-description.md`

### Change Types

| Type | Use For |
|------|---------|
| `feat` | New features |
| `fix` | Bug fixes |
| `refactor` | Code restructuring without behavior change |
| `chore` | Maintenance tasks, dependency updates |
| `docs` | Documentation changes |

## Template

Create a new file with this structure:

```markdown
# [type]: Brief description

**Date**: YYYY-MM-DD
**Type**: feat | fix | refactor | chore | docs

## Summary

What changed and why.

## Files Changed

- `path/to/file.ts` - what changed

## Verification

How to verify this change works:
1. Step 1
2. Step 2

## Related

- Issue: #XXX (if applicable)
- ADR: NNNN (if applicable)
```
